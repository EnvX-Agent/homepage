FAQ

How is EnvX different from a coding agent?

EnvX is repo‑first, not editor‑first. We focus on turning repositories into usable capabilities: setup, data/weights, parameterized tasks, UI + API/SDK, and A2A composition. Coding agents focus on code generation inside an editor.

What does "one‑click setup" include?

Dependency resolution (system & language), data/model weight fetching, env‑var/secret wiring, driver checks (e.g., CUDA), and a reproducible container/VM. Results are cached for fast re‑runs.

What product formats does EnvX offer?

Currently available as a VS Code extension that integrates seamlessly into your existing development workflow. We're also planning standalone desktop applications and web-based interfaces to make EnvX accessible across different development environments.
